## chord-parsing
Implemented Root, Accidental, NoteLetter, Quality, and Chord rules in grammar.pegjs. Added Chord type and Quality union type to types.ts. Exported parseChord() from parser.ts. Added --allowed-start-rules flag to build:grammar so Chord can be used as an entry point. 12 acceptance tests covering C, F#, Bb (major), Am, C#m, Bbm (minor), G7, Bb7, F#7(dominant7), and rejection of Cm7, CM7, Cdim.

## bar-parsing
Added Bar and BarTail rules to grammar.pegjs, with a _ whitespace rule for optional spaces and tabs around the chord. Added Bar type to types.ts. Exported parseBar() from parser.ts. Extended --allowed-start-rules to include Bar. 6 acceptance tests: standard spacing, no spaces, extra spaces, empty bar rejection, two-chord rejection.

## row-parsing
Added Row rule that opens with | then matches one or more BarTail groups (chord + closing |), avoiding double-consumption of shared barlines. Added Row type to types.ts. Exported parseRow() from parser.ts. Extended --allowed-start-rules to include Row. 5 acceptance tests: four-bar row, single-bar row, quality correctness across bars, missing opening barline rejection, missing closing barline rejection.

## frontmatter-parsing
Added FrontMatter, FrontMatterField, FrontMatterValue, and Newline rules to grammar.pegjs. Front matter is enclosed in --- delimiters; fields are parsed as generic key: value pairs and then validated in the action. The key field is validated against the 17 chromatic note spellings (C through B with sharps and flats); invalid values are rejected via Peggy's error() function. Added FrontMatter type to types.ts. Exported parseFrontMatter() from parser.ts. 7 acceptance tests: title + key, title only, key only, empty front matter, all 17 valid keys, invalid key (H), mode suffix rejection (Am).

## song-parsing
Added Song and SongBody rules to grammar.pegjs. Song combines optional FrontMatter with zero or more Rows; blank lines between rows are filtered out in the SongBody action. Added Song type to types.ts. Exported parseSong() from parser.ts. Extended --allowed-start-rules to include Song. 4 acceptance tests: no front matter, front matter + multiple rows, blank line handling, front matter with no rows.

## text-renderer
Created src/renderers/text.ts with a TextRenderer class. renderChord() maps quality back to suffix ('' / 'm' / '7'). renderRow() joins bars as | chord | chord |. renderFrontMatter() emits a --- block only when title or key is non-null. The render() method joins front matter and rows with newlines. 7 tests: front matter output, front matter omission, spacing normalisation, four-bar row on one line, multi-row layout, and two round-trip tests (parse → render → parse produces equal AST).

## key-detector-cat3
Extracted score computation into a `computeScore` helper. Added `breakRelativeTie(major, minor, chords)` which applies ordered heuristics: (1) V7 of harmonic minor present → minor; (2) V7 of major present → major; (3) last-chord tonic match; (4) first-chord tonic match; (5) prefer major. In `detectKey`, after finding the best-scoring key, check if its relative major/minor scores within 1 point (the quality-bonus asymmetry for the I chord of a major key vs III=augmented of its relative minor creates a 1-point gap); if so, invoke tiebreaking. Also added `dom7` and `hd` helpers to the test file and wrote all 7 Category 3 tests (T3-a through T3-g). All 75 tests pass.

## key-detector-cat1
Created src/theory/keyDetector.ts with detectKey(chords: Chord[]): string | null. Scoring assigns 1 point per diatonic chord root (by pitch class) plus a quality bonus point when the chord quality matches the expected diatonic scale degree quality (major keys use major/minor/major/major/minor pattern; harmonic minor keys use minor/halfDiminished/minor/major/major pattern). The quality bonus breaks ties that pure diatonic-root counting cannot resolve (e.g. C major vs G major for [G, D, Em, C]). dominant7 is treated as matching major for quality purposes. Notes unrecognised by rootToPitchClass (E#, B# in key note sets) are silently skipped. Created src/theory/keyDetector.test.ts with 5 Category 1 tests (T1-a through T1-e). All 68 tests pass.

## keys
Created src/theory/keys.ts with: a KeyInfo interface (notes array + optional relative field), a KEYS record of 22 entries (12 major + 10 harmonic minor, excluding G#m and Ebm which require double accidentals), and diatonicNotes() which returns a ReadonlySet<string> for a given key or throws for unknown keys. Harmonic minor keys use the natural minor scale with the 7th degree raised (e.g. Am → G#, C#m → B#, F#m → E#). Relative major/minor links are stored for all pairs that fall within the 22-key set. Created src/theory/keys.test.ts with 9 tests. All 63 tests pass.

## key-detector-cat6
Added optional `declaredKey?: string | null` parameter to `detectKey`. The rule: if declaredKey scores > 0 (any diatonic overlap), return it unchanged; only override it if it scores zero. Also refactored `detectKey` to accumulate all scores into a Map upfront, so the relative-tiebreak lookup reuses cached values instead of recomputing. Added 4 Category 6 tests (T6-a through T6-d): single-chord best-guess, declared-key preservation, declared-key correction when score is zero, and three-chord unambiguous identification. All 82 tests pass.
