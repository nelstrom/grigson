## detect-key-ending-wins
Added an ending-key-wins rule to detectKey in src/theory/keyDetector.ts. When a progression ends with a V7→tonic cadence (dominant7 penultimate chord resolving to a major or minor tonic as the final chord), and the resolved tonic is the root of a candidate key that scored within 1 point of the overall best-scoring key, that cadence key is promoted as the result. The rule requires an authentic V7→I cadence to fire (penultimate chord quality = dominant7, and its root must be exactly a fifth above the candidate key's tonic) — this prevents simple diatonic final chords from incorrectly overriding a clear winner. Added 3 tests in a new 'ending-key-wins' describe block: the Whisper Not A-section [Cm, D7, Gm, A7, Dm] → 'Dm' (new), and two confirmations that existing logic is preserved. All 136 tests pass.

Added `runTranspose()` which reads input from a file argument or stdin, dispatches to `transposeSongToKey` (for `--to`) or `transposeSong` (for `--raise`/`--lower`), and writes normalised output to stdout. Used `--raise <n>` and `--lower <n>` (both positive integers) instead of `--semitones <n>` (positive or negative) to avoid a minimist flag-parsing ambiguity where `-3` is treated as the flag `-3` rather than the value of `--semitones`. Exits with code 1 if more than one of the three flags is provided or if none is provided. Added 6 acceptance tests covering all scenarios. Updated documentation/cli.md and the prd.json detail/steps. All 176 tests pass.

## chord-qualities-parse-render
Added four new Quality union members to src/parser/types.ts: 'diminished', 'maj7', 'min7', 'dim7'. Extended the Peggy grammar (grammar.pegjs) with new alternatives in PEG-safe order: m7b5 > maj7 > M7 > dim7 > m7 > dim > m > 7 > "" — longer alternatives precede shorter ones to avoid premature matches. CM7 and Cmaj7 both produce quality 'maj7'; canonical output is always 'maj7'. Extended TextRenderer.renderChord() with a switch statement covering all eight qualities (dim → 'dim', maj7 → 'maj7', min7 → 'm7', dim7 → 'dim7'). Updated the old "unsupported qualities" test block — Cm7, CM7, and Cdim are now supported; replaced with tests for truly unsupported suffixes (sus4, aug). Added 9 parser tests covering the four new qualities (including CM7 normalisation to maj7) and 9 renderer tests (render + round-trip for each new quality). All 194 tests pass.

## normalise-harmonic
Rewrote normaliseSection in src/theory/normalise.ts to call analyseHarmony and use each chord's currentKey for enharmonic spelling. The updated normaliseChord now takes two pcToNote maps: (1) homePCToNote from the detected homeKey — used first for any pitch class that is diatonic to the homeKey (handles enharmonic corrections like C#→Db in Db major); (2) currentPCToNote from the analyseHarmony-assigned currentKey — used for pitch classes not in homeKey (handles borrowed chords like Bb in C major being spelled from F major's note set, or Eb7 in a 2-5-1 to Ab being spelled from Ab's note set). Removed the old FLAT_SPELLING constant and preferredNonDiatonicSpelling next-chord heuristic entirely. All existing Cat 2, 5, 8 tests continue to pass. All 164 tests pass.

## chord-qualities-tetrad-tests
Added `maj7`, `min7`, and `dim7` chord helper functions to src/theory/keyDetector.test.ts. Added a new 'detectKey — tetrad quality disambiguation' describe block with 6 tests demonstrating that tetrad qualities resolve ambiguous cases that triads cannot: (1) Cmaj7/Gmaj7 alternation → G (Gmaj7 misses C's V-set {major, dominant7}); (2) Cmaj7/G7 alternation → C (G7 misses G's I-set {major, maj7}); (3) full ii-V-I-vi in C with tetrads (Dm7, G7, Cmaj7, Am7 → C); (4) ii-V-I in D (Em7, A7, Dmaj7 → D); (5) iiø-V7-i in Am with tetrads (Bm7b5, E7, Am7 → Am via V7-of-minor tiebreak); (6) Am harmonic minor with G#dim7 VIIdim7 as dominant substitute → Am via last-chord tonic tiebreak. The PRD step listed Gdim7 but the comment specified G#dim7 (the diatonic VIIdim7 of Am); used G#dim7 since Gdim7 is non-diatonic to Am and would break the threshold. No implementation changes were needed — the existing Set<Quality>-based scorer handles all cases correctly. All 200 tests pass.

## validator-core
Created src/validator.ts exporting DiagnosticRange, Diagnostic interfaces and a validate(source: string): Diagnostic[] function. The function calls parseSong() and returns [] for valid input. For Peggy parse errors it uses an isPeggyError type guard to check for the location property, then maps the 1-indexed Peggy line/column to 0-indexed LSP-style range values (subtracting 1 from each). Non-Peggy errors fall back to a zero range with String(e) as the message. Exported validate, Diagnostic, and DiagnosticRange from packages/grigson/src/index.ts. Added src/validator.test.ts with 7 tests: empty file valid, minimal chart valid, unsupported suffix returns one error, garbage input returns one error, range values are non-negative integers, valid chart with front matter returns [], and error location reflects the correct line for multi-line input. All 215 tests pass.

## cli-validate
Implemented the `grigson validate` subcommand in src/cli.ts. Added `runValidate()` which reads from one or more file arguments or stdin (labelled `<stdin>`), calls `validate()` on each, and reports diagnostics. Default text output uses the standard Unix linter format `<file>:<line>:<character>: <severity>: <message>` with 1-indexed line and character numbers. `--format json` emits a JSON array of `{ file, line, character, severity, message }` entries. Exits with code 0 when all inputs are valid, code 1 when any error is found. Registered `validate` in the SUBCOMMANDS tuple, dispatch switch, and all help strings. Added 5 acceptance tests in `src/cli.test.ts` covering: valid file exits 0, invalid file exits 1 with filename+line, stdin reading, JSON empty array on valid input, and JSON array with typed error entry on invalid input. Updated `documentation/cli.md` with a full `grigson validate` section including text and JSON output format descriptions and usage examples. All 220 tests pass.

## language-server
Created packages/language-server/ as a new workspace package (already covered by pnpm-workspace.yaml's 'packages/*' glob). package.json sets name 'grigson-language-server' with bin pointing to ./dist/server.js. Dependencies: vscode-languageserver, vscode-languageserver-textdocument, and grigson (workspace:*). Because vscode-languageserver is a CommonJS package and grigson is ESM, server.ts uses a dynamic import() for grigson at startup and CommonJS module output via tsconfig.json (module: CommonJS, moduleResolution: Node). The build script compiles with tsc and prepends a #!/usr/bin/env node shebang via a Node.js one-liner. The server calls validate() on textDocument/didOpen and textDocument/didChange events and sends diagnostics via connection.sendDiagnostics(). Smoke-tested by sending an LSP initialize message via stdin and confirming the capabilities response. Created documentation/language-server.md with usage instructions and per-editor configuration for VS Code, Neovim, Helix, and Emacs. All 220 existing tests continue to pass.
